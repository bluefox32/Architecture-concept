# エッジ分散アーキテクチャ プロトタイプ

次世代エッジ分散アーキテクチャの設計思想をPythonで示したプロトタイプ実装です。  
**アプリサービス × 通信 × エッジサーバー**を統合したビジネスモデルの技術的基盤を検証します。

---

## 概要

クラウド中心の設計からローカル分散クラスターへの転換を図り、以下の特性を同時に実現します。

- **データの中身を見ない設計** — HashとメタデータのみでプライバシーとPを構造的に保護
- **オフライン耐性** — 切断中も自律動作し、再接続時にRootまで自動同期
- **スコアベースのサイドロード** — 信頼スコア＋コスト差分で安全性と経済合理性を同時評価
- **多層ミラーリングバックアップ** — RAID的な冗長性をソフトウェアで実現
- **広告×カウントフリーの統合収益モデル** — エッジ化でインフラコストを逓減させながら収益を最大化

---

## アーキテクチャ構成

```
ユーザー端末（エッジ）
      │ ローカル処理
公衆WiFiスポット
      │ 近距離高速
エッジサーバー
      │ メタデータのみ
センター（アーカイブ）
```

### レイヤー構造

| レイヤー | 役割 | 技術的根拠 |
|----------|------|-----------|
| アプリ内組み込みDB | オフライン継続・差分管理 | SQLite WAL / RocksDB |
| ローカルクラスター | 主力処理・RAID冗長化 | Proxmox / Ceph / ZFS |
| エッジサーバー | 近距離同期・トラフィック分散 | k3s / JupyterHub |
| センターサーバー | 長期アーカイブのみ | コールドストレージ |

## 実装カテゴリ

| カテゴリ | クラス | 概要 |
|----------|--------|------|
| CAT-1 | データ構造・定数 | Node / Package / DataEntry / WALRecord |
| CAT-2 | Hash認証レイヤー | Merkle Tree / SHA-256 / 署名検証 |
| CAT-3 | ノード管理 | レイテンシ計測・登録・状態管理 |
| CAT-4 | スコアエンジン | 信頼スコア + コスト差分の統合評価 |
| CAT-5 | CRDT同期エンジン | Vector Clock / 差分マージ / WAL |
| CAT-6 | サイドロードマネージャー | 対象分類・ロード判定 |
| CAT-7 | バックアップエンジン | スナップショット / レプリケーション |
| CAT-8 | エッジルーター | 最適ノード選択 / 接続モード判定 |
| CAT-9 | APIレイヤー | 全コンポーネントの統合インターフェース |

### 処理の命名規則

```
branch_*  : 分岐処理（判定・分類・条件評価）
process_* : 主要処理（実行・変換・状態変更）
```

### 実行結果（デモシナリオ）

```
[1] システム起動          — ノード登録・レイテンシ初期計測
[2] データ書き込み        — Hash生成・WAL記録・Merkle Tree構築
[3] サイドロード評価      — スコアエンジンによる自動実行/審査/拒否
[4] オフライン→再接続    — WAL蓄積→CRDT自動マージ
[5] Root整合性確認        — Merkle RootHashの一致検証
[6] バックアップ           — 多層レプリケーション・スナップショット
[7] ビジネス収支レポート  — ユーザー規模別の月次試算
```

## スコアエンジンの評価モデル

```
TrustScore =
    署名検証スコア    × 0.30
  + 出所信頼度        × 0.25
  + 過去実績          × 0.20
  + リソース影響度    × 0.15
  + コスト差分        × 0.10
```

スコアに応じた自動判定：

| スコア | 判定 | アクション |
|--------|------|-----------|
| 0.75 以上 | HIGH | 自動実行 |
| 0.45〜0.75 | MEDIUM | 審査待ち |
| 0.45 未満 | LOW | 拒否 |

## 整合性保証の仕組み

```
切断中：Availability優先（ローカルで動き続ける）
再接続：Consistency優先（Rootと整合性を取る）

Vector Clock → 差分特定
CRDT        → 自動マージ（競合を人手なしに解決）
Merkle Tree → RootHash一致でEnd-to-End整合性確認
```

## 設計上の注意点

- 本実装はプロトタイプです。本番環境では以下の対応が必要です。
  - レイテンシ計測をICMP ping / TCPソケットに置き換え
  - 署名検証をRSA / ECDSAに置き換え
  - ネットワーク転送部分の実装（現在はメモリ内模擬）
  - 耐量子暗号（PQC）への将来的な移行計画
